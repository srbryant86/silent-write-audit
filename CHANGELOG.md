# Changelog

All notable changes to this project will be documented in this file.

## [0.3.0] — 2026-05-07

### Added
- **Prisma adapter.** Validates Prisma client call sites against `schema.prisma`. Covers `create`, `update`, `upsert`, `delete`, `findUnique`, `findFirst`, `findMany`, `findFirstOrThrow`, `findUniqueOrThrow`, `updateMany`, `deleteMany`, `count`, `aggregate`, and `createMany`. Cross-references payload keys (`data` / `create` / `update`), `where` keys, and `select` keys against the model's declared scalar fields.
- **Compound-unique-key recognition.** Synthetic `where`-clause keys generated by `@@id([a, b])` and `@@unique([a, b])` directives are treated as valid. Supports all four Prisma syntactic forms: `[a, b]`, `[a, b], name: "k"`, `fields: [a, b]`, and `fields: [a, b], name: "k"`.
- **Multi-schema monorepo handling.** Auto-detects sibling `schema.prisma` files under scan paths (e.g. `packages/platform/examples/base/prisma/schema.prisma`) and skips Prisma checks under those subtrees. Boundaries are reported on startup so the user can confirm scope.
- **CLI flags:** `--prisma-schema PATH` (custom schema path), `--skip-supabase` (Prisma-only mode), `--skip-prisma` (Supabase-only mode). Both schema sources can run concurrently against the same codebase when both are configured.
- **Config:** `prismaClientNames` array for custom Prisma client identifiers (default: `["prisma", "db", "tx", "client"]`).
- README "Prisma adapter (v0.3+)" section with usage examples, supported patterns, and compound-key explainer.

### Validation
- Smoke-tested against [Cal.com](https://github.com/calcom/cal.com) (4367 TypeScript files, 100 Prisma models, 1 sub-schema). Initial scan produced 8 findings; after compound-unique-key support and multi-schema detection, dropped to 1 finding (a real bug — `prisma.watchlist.create` with phantom `createdById`). False-positive rate 0/1 on this corpus.

### Why v0.3 instead of waiting for demand signal
- Earlier roadmap gated v0.3 (Drizzle / Prisma) on first user request. While testing the cold-outreach hook on candidate buyer codebases (PostHog, Cal.com, Mintlify), the empirical buyer cohort turned out to be ORM-using (Prisma / Django ORM / closed-source) rather than raw Supabase-JS. The Prisma adapter unblocks the audit-as-bait pre-call hook for the largest sub-segment of the bench. Drizzle remains demand-gated.

## [0.1.0] — 2026-05-06

Initial public release. Extracted from CertNode's internal audit tooling.

### Added
- Static analysis of `.update()`, `.upsert()`, `.insert()`, `.select()`, and PostgREST filter chains (`.eq()`, `.neq()`, `.gt()`, etc.) against the live database schema.
- Phantom-column detection: identifies columns referenced in code that don't exist on the target table.
- Schema source: Supabase Management API (with 24h local cache) — direct `pg_dump` and DATABASE_URL support coming in v0.2.
- JSONB path recognition (`metadata->>field`, `data->'a'->>'b'` resolve to base column).
- Inline allowlist via `// silent-write-audit-ignore` comment.
- JSON output mode (`--json`) for CI integration.
- Staged-only mode (`--staged`) for pre-commit hooks.
- Severity tagging by user-defined critical-table list (`.silent-write-audit.json`).
- Example pre-commit hook that runs the audit on staged TypeScript changes.

### Known limitations
- Schema source: Supabase Management API only. PostgreSQL direct connection (DATABASE_URL) and `pg_dump` schema files come in v0.2.
- ORM detection: only PostgREST/Supabase JS client patterns. Drizzle/Prisma/TypeORM auditing is on the roadmap (open an issue if you need it).
- Spread operators in payloads (`{...obj}`) are skipped (can't be statically analyzed).
- Joined-relation column filters (`.eq('related.col', v)`) are skipped to avoid false positives.

### Origin

This tool was built to fix our own production bugs. Pointed at our own codebase (CertNode, ~80k lines of TypeScript), it found ~50 silent-write bugs in a single night, including dispute outcomes that hadn't been persisting for weeks, billing flags that never flipped after deauthorization, and a GDPR redaction handler that was a silent no-op. The pre-commit hook variant has been blocking new bugs of this class on every commit since 2026-05-05.
